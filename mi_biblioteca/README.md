## Breve resumen de la estructura del caso de estudio (documentación)

El presente caso de estudio se desarrolla siguiendo una metodología estructurada en cuatro fases principales de la Programación Orientada a Objetos (POO), las cuales permiten transformar un problema del mundo real en una solución de software funcional.

**1. Requerimientos**
En esta fase se analiza la narrativa del problema para identificar las necesidades del sistema. Se determinan los actores principales, las funcionalidades requeridas y las restricciones del sistema. Además, se identifican sustantivos (posibles clases y atributos) y verbos (posibles métodos).

**2. Análisis**
Se aplica el proceso de abstracción para definir las entidades principales del sistema, sus atributos esenciales y comportamientos. También se identifican las relaciones entre clases, como herencia, asociación, agregación y composición.

**3. Diseño**
Se elabora el diagrama de clases UML, donde se representan las clases con sus atributos, métodos, niveles de visibilidad y relaciones. Este diagrama sirve como modelo base para la implementación del sistema.

**4. Desarrollo**
Se implementa el sistema en Python, traduciendo el diseño UML a código funcional. Se aplican los pilares de la POO:
- Abstracción
- Encapsulamiento
- Herencia
- Polimorfismo

El sistema desarrollado permite gestionar libros, estudiantes y préstamos dentro de una biblioteca universitaria, incluyendo funcionalidades como registro, préstamo, devolución y consulta de información.
