## Breve resumen de la estructura del caso de estudio (documentación).
El caso de estudio describe el desarrollo de un sistema básico para la gestión de una biblioteca siguiendo las etapas de la Programación Orientada a Objetos. Primero, se identifican los requerimientos a partir del análisis del problema, determinando las funcionalidades, actores y elementos clave. Luego, en la fase de análisis, se definen las entidades principales, sus atributos, comportamientos y relaciones mediante abstracción. En la etapa de diseño, se elabora el diagrama de clases UML que sirve como guía estructural del sistema. Finalmente, en la fase de desarrollo, se implementa la solución en Python aplicando los principios de la POO, dando como resultado un sistema funcional para gestionar libros, estudiantes y préstamos.
